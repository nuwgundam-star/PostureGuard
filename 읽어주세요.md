## 0. 먼저 — 깃허브 폴더는 뭐가 다른가?

**같은 코드 맞아.** 근데 깃허브에는 **실행에 필요한 게 빠져 있어.**

`.gitignore` 때문에 아래가 깃허브에 안 올라가거든:

| 빠진 것 | 원래 역할 |
|---|---|
| `backend\venv\` | 파이썬 라이브러리 전체 (679MB, 용량 때문에 깃 제외) |
| `frontend\dist\` | **대시보드 화면 그 자체** (빌드 결과물) |
| `backend\.env` | 설정 파일 |
| `node_modules\` | 프론트 빌드 도구 |

그래서 깃허브에서 받으면 **폴더 이름은 똑같은데 실행하면 아무것도 안 뜬다.**
네가 "파일이 없다"고 한 게 정확한 관찰이었어. **이 폴더(지금 받은 것)를 써줘.** 깃허브 건 무시.

## 1. 두 가지 방법 — 편한 걸로

### 방법 A. 영상만 쓰기 (제일 안전) ⭐ 추천
`시연영상.mp4` 를 발표 슬라이드에 그냥 삽입.
**설치·실행 아무것도 필요 없음.** 32초, 1600x900.

### 방법 B. 라이브 실행 (백업용)
1. **Python 3.11 설치** — [python.org](https://www.python.org/downloads/release/python-3119/)
   - 설치 첫 화면에서 **"Add python.exe to PATH" 체크** (제일 중요)
   - ⚠️ **반드시 3.11.** 3.12·3.13 은 안 돼 (라이브러리 호환 안 됨)
2. `start.bat` 더블클릭
3. 검은 창 2개가 뜸 → **닫지 말 것**
4. 10초 뒤 브라우저 자동 오픈 (안 열리면 주소창에 `localhost:3000`)
   - ⚠️ **반드시 `localhost`.** `127.0.0.1` 로 치면 버튼이 안 먹혀 (CORS 차단)
5. 우상단 "연결됨" 확인 → 상단 버튼(양호/주의)으로 시연

> `start.bat` 이 Python 3.11 을 못 찾거나 라이브러리 로딩에 실패하면
> **에러 메시지를 띄우고 멈춰.** 조용히 실패하진 않으니까 창 내용을 읽어보면 돼.

## 2. ⚠️ 영상 화면 ≠ 라이브 화면 (발표 때 헷갈리지 말기)

둘이 **생김새가 달라.** 알고 가야 당황 안 해.

| | 화면 | 내용 |
|---|---|---|
| **시연영상.mp4** | 좌: 실제 웹캠 / 우: **파란 홀로그램 사람** | 영상 전용으로 따로 렌더한 것 |
| **라이브 (start.bat)** | React 대시보드 | 히트맵·그래프 중심의 실제 제품 화면 |

**측정 원리·점수 계산은 완전히 동일해.** 보여주는 방식만 달라.
발표에선 **영상을 메인**으로 쓰고, 라이브는 "실제로 돌아갑니다" 보여줄 때만 쓰면 깔끔해.

## 3. 영상에 나오는 것 설명 (발표 대본용)

- **좌측**: 실제 웹캠 영상 (내가 앉아서 자세 바꾸는 장면)
- **우측**: 디지털 트윈 — 웹캠에서 뽑은 **관절 좌표만으로** 3D 인체를 구동
  - 영상은 서버로 안 보내고 **숫자(좌표)만** 전송 → 프라이버시 보호가 설계 포인트
  - **문제 부위에만** 조준 표시가 뜸 (위험=빨강 / 주의=주황)
- **하단 HUD**
  - `거북목 /20` — 목이 앞으로 나온 정도
  - `어깨 틀어짐 /10` — 좌우 어깨 높이 차
  - `RISK /100` — 종합 위험도 (NOTICE < 40 ≤ WARNING < 70 ≤ DANGER)

점수 산식 상세는 `docs\위험도_측정_구조와_원리.md` 참고 (깃허브에 있음).

## 4. 안 될 때

| 증상 | 해결 |
|---|---|
| `start.bat` 이 "Python 3.11 을 찾지 못했습니다" | Python 3.11 설치 + PATH 체크. 설치 후 재실행 |
| 버전은 맞는데 라이브러리 에러 | 3.11 이 아닌 다른 버전이 먼저 잡힌 것. 창에 찍힌 버전 확인 |
| 브라우저는 열리는데 화면이 비어있음 | 주소가 `localhost:3000` 인지 확인 (`127.0.0.1` ❌) |
| "연결 끊김" | Backend 검은 창 확인 → 창 다 닫고 `start.bat` 재실행 |
| 다 안 됨 | **방법 A(영상)로 진행.** 심사에 지장 없음 |

## 5. 웹캠 없어도 됨

웹캠이 없거나 인식 안 돼도 **서버는 안 죽어.** 더미 데이터로 대시보드가 그대로 돌아가.
압력방석(FSR)도 이번 시연엔 안 들어가 있어 — **웹캠 기반 자세 판단만** 시연 범위야.
(방석은 가짜 데이터로 흉내내면 심사에서 문제 되니까 일부러 뺐어.)
